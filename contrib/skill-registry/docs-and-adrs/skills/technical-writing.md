# Technical docs & ADRs

- **README:** what it does, how to install/run, link to deeper docs; keep the first screen scannable.
- **ADRs (Architecture Decision Records):** short title, context, decision, consequences; one decision per file; name with index e.g. `0001-use-sqlite.md`.
- **User-facing copy:** imperative for steps (“Run …”), avoid vague “just” / “simply”.
- **Accuracy:** every code block or path should exist in repo or be marked hypothetical.
